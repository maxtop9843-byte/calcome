## Formula

Document applicable formulas, variables, assumptions, precision, rounding rules, and authoritative sources. Write `Not applicable` when the feature has no formula.

## Validation

- Describe accepted ranges and formats.
- Define clear user-facing error behavior.

## Edge Cases

- List boundary, empty, invalid, precision, and extreme-value cases.

## Mobile UX

Describe the mobile-first task flow, layout, touch behavior, and content priority.

## Desktop UX

Describe progressive enhancements for wider viewports without changing the core flow.

## Accessibility

Define semantics, labels, keyboard behavior, focus management, announcements, contrast, reduced motion, and touch targets.

## SEO

Define search intent, metadata, canonical behavior, visible explanatory content, and applicable structured data.

## Related Calculators

- List genuine relationships, or write `None at launch`.

## Testing Requirements

- Define formula and validation unit tests when applicable.
- Define component, accessibility, integration, and end-to-end coverage appropriate to risk.

## Implementation Plan

Map the approved requirements to existing architecture, shared components, test coverage, and verification. Do not include unrelated refactoring.
