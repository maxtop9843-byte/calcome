# Deposit Calculator

Status: Implemented

## Purpose

한 번에 맡기는 예금 원금이 고정 연이율과 기간에 따라 얼마나 늘어나는지 단리 또는 월복리로 계산하고, 세전 이자와 예상 세금, 세후 만기액을 명확히 구분한다. 특정 은행 상품의 실제 지급액을 재현하지 않고 가입 전 시나리오 비교를 돕는다.

## Requirements

- 예치 금액, 기간, 연 이자율, 이자 방식과 세금 옵션을 입력받는다.
- 기간은 개월 또는 년으로 받고 내부에서 개월 수로 정규화한다.
- 단리와 월복리를 지원한다.
- 비과세, 일반과세 15.4%, 사용자 지정 간이 세율을 지원한다.
- 원금, 세전 이자, 예상 세금, 세후 이자, 세전·세후 만기액과 실효 수익을 같은 계산 결과에서 제공한다.
- Decimal.js로 계산하고 표시 전까지 반올림하지 않는다.
- 변동금리, 우대금리, 중도해지, 수수료와 특정 금융상품 규칙은 계산하지 않는다.

## Inputs

| Input            | Type   | Unit/Format                 | Required | Default      | Validation              |
| ---------------- | ------ | --------------------------- | -------- | ------------ | ----------------------- |
| 예치 금액        | 정수   | 원                          | 예       | 10,000,000원 | 10,000원~1조원          |
| 예치 기간        | 정수   | 개월/년                     | 예       | 1년          | 1~1,200개월             |
| 연 이자율        | 소수   | %                           | 예       | 3.5%         | 0~100%                  |
| 이자 방식        | 열거형 | 단리/월복리                 | 예       | 단리         | 지원 값만 허용          |
| 세금 옵션        | 열거형 | 비과세/일반과세/사용자 지정 | 예       | 일반과세     | 지원 값만 허용          |
| 사용자 지정 세율 | 소수   | %                           | 조건부   | 빈 값        | 사용자 지정일 때 0~100% |

## Outputs

| Output      | Unit | Description                          |
| ----------- | ---- | ------------------------------------ |
| 원금        | 원   | 입력한 예치 금액                     |
| 세전 이자   | 원   | 선택한 이자 방식으로 계산한 총 이자  |
| 예상 세금   | 원   | 세전 이자에 적용 간이 세율을 곱한 값 |
| 세후 이자   | 원   | 세전 이자에서 예상 세금을 뺀 값      |
| 세전 만기액 | 원   | 원금 + 세전 이자                     |
| 세후 만기액 | 원   | 원금 + 세후 이자                     |
| 실효 수익   | %    | 원금 대비 세후 이자 비율             |

## Assumptions and Formula

`P`는 예치 원금, `r`은 연 이자율의 소수값, `m`은 전체 예치 개월 수다.

- 단리 세전 이자: `P × r × m / 12`
- 월복리 세전 만기액: `P × (1 + r / 12)^m`
- 월복리 세전 이자: `세전 만기액 - P`
- 예상 세금: `세전 이자 × 적용 간이 세율`
- 세후 만기액: `P + 세전 이자 - 예상 세금`
- 실효 수익: `세후 이자 / P × 100`

월복리는 명목 연이율을 12로 나눈 월 이자율을 사용한다. 0%에서는 이자와 세금이 0원이고 세전·세후 만기액은 원금과 같다. 계산은 Decimal.js의 고정밀 값을 유지하며 원 단위 표시는 반올림 경계에서만 처리한다.

일반과세 15.4%는 거주자의 일반 이자소득에 대한 소득세 원천징수 14%와, 그 소득세의 10%인 개인지방소득세 1.4%를 합친 간이 세율이다. 실제 과세는 가입자, 상품과 비과세·과세특례 적용에 따라 달라질 수 있다. 이 세율과 근거는 Savings 계산기에서 검증된 공용 상수를 재사용한다.

## Validation

- 금액은 원 단위 정수만 허용하며 빈 값, 음수, 지수 표기, 잘못된 쉼표와 범위 밖 값을 거부한다.
- 기간은 1 이상의 정수이며 최대 1,200개월이다.
- 이자율과 사용자 지정 세율은 0~100%의 유한한 숫자만 허용한다.
- 사용자 지정 세율은 해당 세금 옵션에서만 필수다.
- 제출 실패 시 입력값을 유지하고 연결된 오류를 표시하며 첫 오류 입력으로 포커스를 옮긴다.

## Edge Cases

- 1개월 단리와 월복리를 정확히 계산한다.
- 100년(1,200개월)에서도 유한하고 일관된 결과를 반환한다.
- 0% 이율, 비과세와 사용자 지정 0% 세율에서 세금이나 수익 계산 오류가 없다.
- `세전 만기액 = 원금 + 세전 이자`와 `세후 만기액 = 원금 + 세후 이자`가 항상 일치한다.

## Accessibility

모든 입력은 영구 라벨, 도움말과 연결된 오류 설명을 가진다. 키보드로 모든 컨트롤을 조작할 수 있고 포커스가 시각적으로 드러난다. 오류 요약은 `role=alert`, 완료 메시지는 `aria-live=polite`를 사용한다. 모바일에서 입력과 결과를 한 열로 제공하고 넓은 화면에서 나란히 배치한다. 정보는 색만으로 전달하지 않는다.

## SEO

경로는 `/finance/deposit`이다. 고유 한국어 title과 description, canonical, Open Graph, Twitter/X 메타데이터를 제공한다. 보이는 breadcrumb와 일치하는 `BreadcrumbList`, 페이지의 실제 설명과 일치하는 `WebPage`만 구조화 데이터로 제공한다.

## Architecture

- `src/features/deposit/`에서 types, constants, validation, calculation, formatting, client UI를 분리한다.
- 서버 route는 metadata, 구조화 데이터, 정적 한국어 콘텐츠와 client boundary를 조합한다.
- 공용 일반 이자소득 간이 세율은 `src/lib/finance/tax.ts`에 두고 Savings와 Deposit에서 함께 사용한다.
- 기존 `createPageMetadata`, published-calculator registry와 카드·sitemap 구성을 재사용한다.
- Compound Interest, Loan, Savings 계산 공식은 변경하지 않는다.
